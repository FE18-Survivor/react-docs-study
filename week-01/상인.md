# Week 01

## library vs framework

> React is a JavaScript library for rendering user interfaces (UI)
>
> React는 사용자 인터페이스(UI)를 렌더링하기 위한 자바스크립트 라이브러리다.

둘을 구분하는 핵심은 제어권을 누가 가지고 있는지이다. 라이브러리는 개발자가 필요한 기능을 호출하여 사용하며, 제어권이 개발자에게 있다. 반면에 프레임워크는 애플리케이션 구조와 흐름을 제어하는 뼈대이다. 일정한 구조와 규칙이 있고, 그 안에서 개발자가 코드를 작성하는 방식이다. 프레임워크에게 제어권이있고 개발자는 필요한 부분만 구현한다.

리액트는 UI 구성에 집중하고, 라우팅(react-router), 상태관리(redux,zustand, tanstack-query), 데이터 처리 등은 별도 도구를 조합하여 개발자가 자유롭게 사용하기에 **제어권이 개발자**에게 있다고 해서 라이브러리로 분류한다고 이해했다.

## Your First Component

React 컴포넌트는 자바스크립트 함수이며, HTML과 비슷한 JSX 문법을 추가해 UI를 정의함

```javascript
export default function Profile() {
  return (
    <img
      src="https://i.imgur.com/MK3eW3Am.jpg"
      alt="Katherine Johnson"
    />
  )
}

// 개인적으로는 아래 방식 선호
const Profile = () => {
  return (
    <img
      src="https://i.imgur.com/MK3eW3Am.jpg"
      alt="Katherine Johnson"
    />
  )
}

export default Profile;
```

### 주의할 점들

1. 첫글자는 대문자

```javascript
// HTML 태그 img 를 사용함
return <img />;

// 정의해놓은 Img 컴포넌트를 사용함
return <Img />;
```

2. 반환문 규칙

```javascript
// 한 줄로 처리
return <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />;

// 줄이 달라지면 괄호를 사용해야 한다
return (
  <div>
    <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  </div>
);
```

3. 컴포넌트 내에서 컴포넌트를 정의하지 않도록 한다.

```javascript
export default function Gallery() {
  // 🔴 Never define a component inside another component!
  function Profile() {
    // ...
  }
  // ...
}
```

부모 컴포넌트가 리렌더링될 때마다 내부에 정의된 자식 컴포넌트 함수가 "새로운 함수"로 다시 생성되어 리액트가 이전과 다른 컴포넌트로 인식하게 된다. -> state 와 memoization이 초기화, 불필요한 리렌더링

## Importing and Exporting Components

여러 개의 컴포넌트를 한 파일에서 선언할 수 있지만, 파일이 커지면 관리가 힘들 수 있다. 그래서 보통 컴포넌트별로 파일을 나누고, import/export를 사용한다.

```javascript

// Gallery.js
export function Profile() { // named export
  return (
    <img
      src="https://i.imgur.com/QIrZWGIs.jpg"
      alt="Alan L. Hart"
    />
  );
}

export default function Gallery() { // default export
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
// App.js
import Gallery, { Profile } from './Gallery.js';

export default function App() {
  return (
    ...
  );
}
```

## Writing Markup with JSX

React 컴포넌트에서는 JSX를 사용하고, 로직과 마크업을 한 파일에서 관리하여, 수정 및 협업이 편리해진다. 너무나도 큰 장 점으로 더이상 HTML의 태그에 id, class를 부여하지 않아도 된다.

### JSX

JSX는 자바스크립트에서 HTML 비슷한 코드를 직접 쓸 수 있게 해주는 확장된 문법이다.

> React는 JSX를 어떻게 변환하여 렌더링할까?
>
> JSX → React Element → virtual DOM 생성 → Fiber
>
> [React의 렌더링 과정](https://jeongsangin1.tistory.com/entry/%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80%EC%9D%98-DOM-%EA%B3%BC-React%EC%97%90%EC%84%9C%EC%9D%98-Virtual-DOM)

### JSX 규칙

- 닫는 태그를 반드시 붙여야 하며, `self-closing` 태그도 반드시 닫는 형태로 써야 한다.
- `class → className`, `tabindex → tabIndex` 등 대부분 속성명을 camelCase (data-와 aria- 속성만은 HTML과 동일)
- 여러 개의 태그를 반환할 때 반드시 하나의 부모 태그로 감싸기
- 불필요하게 div 등 태그를 늘리지 않고 `<> ... </>` 형태의 fragment로 여러 엘리먼트를 한번에 감쌀 수 있다.
