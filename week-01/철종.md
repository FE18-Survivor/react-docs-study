# Week 01

## Describing the UI

> React는 UI를 렌더링하는 JavaScript 라이브러리 입니다. <br>
> UI(User Interface)는 Buttun, Text, Image와 같은 단위로 구성됩니다.<br>
> React를 사용하면 이러한 UI 단위를 재사용이 가능하고, 중첩 가능한 컴포넌트로 조합할 수 있습니다.<br>
> 웹사이트에서 휴대폰 앱에 이르기까지 화면의 모든것은 컴포넌트로 나눌 수 있습니다.<br>
> 이 장에서는 React 컴포넌트를 만들고, 사용자 정의하고, 조건부로 표시하는 방법을 배웁니다.<br>

## 첫 번째 컴포넌트 [Your First Component](https://react.dev/learn/your-first-component)

> 리액트 애플리케션은 컴포넌트라고 불리는 분리된 UI 요소들로 구성되어 있습니다. <br>
> 리액트 컴포넌트는 마크업을 추가 할수 있는 JavaScript 함수 입니다.<br>
> 컴포넌트는 버튼만큼 작거나 페이지만큼 클 수 있습니다.<br>
> 다음은 세 개의 Profile을 렌더링하는 Gallery 컴포넌트입니다.<br>

> 컴포넌트는 React의 핵심 개념 중 하나입니다.
> 사용자 인터페이스(UI)를 구축하는 기반이 되므로 React 여정을 시작하는 완벽한 곳입니다!.

### 컴포넌트: UI 구성 요소

- 이 마크업은 문서 <article>, 제목 <h1>, 그리고 (축약된) 목차를 순서가 있는 목록 <ol>으로 나타냅니다.
- 이러한 마크업은 스타일을 위한 CSS, 상호작용을 위한 JavaScript와 결합되어 웹에서 보는 모든 사이드바, 아바타, 모달, 드롭다운 등 모든 UI의 기반이 됩니다.

- React를 사용하면 마크업, CSS, JavaScript를 결합하여 앱에서 재사용 가능한 UI 요소인 사용자 지정 "컴포넌트"를 만들 수 있습니다.
- 위에서 본 목차 코드는 모든 페이지에 렌더링할 수 있는 <TableOfContents /> 컴포넌트로 변환할 수 있습니다.
- 내부적으로는 <article>, <h1> 등과 같은 동일한 HTML 태그를 사용합니다.

- HTML 태그와 마찬가지로 컴포넌트를 구성, 정렬 및 중첩하여 전체 페이지를 디자인할 수 있습니다.
- 예를 들어, 지금 읽고 있는 문서 페이지는 React 컴포넌트로 구성되어 있습니다.

### 컴포넌트 정의하기

- 전통적으로 웹 페이지를 만들 때 웹 개발자는 콘텐츠를 마크업한 후 JavaScript를 활용하여 상호작용 기능을 추가했습니다.
- 이 방식은 웹에서 상호작용 기능이 필수적이었던 시절에는 매우 효과적이었습니다.
- 이제는 많은 사이트와 모든 앱에서 이러한 기능은 필수적입니다.
- React는 동일한 기술을 사용하면서도 상호작용성을 우선시 합니다.
- React 컴포넌트는 마크업을 추가할 수 있는 JavaScript 함수입니다.
- 컴포넌트의 모습은 다음과 같습니다(아래 예시를 편집할 수 있습니다).

App.js

```javascript
export default function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}
```

> 1단계 : 컴포넌트 내보내기(Export)

- 내보내기(Export) 기본 접두사는 표준 JavaScript 구문입니다(React에만 국한되지 않음).
- 이를 통해 파일에서 메인 함수를 표시하여 나중에 다른 파일에서 가져올 수 있습니다.
- (가져오기에 대한 자세한 내용은 컴포넌트 가져오기(import) 및 내보내기에서 확인하세요!)

> 2단계 : 함수 정의

- Profile() { } 함수를 사용하여 Profile이라는 이름의 JavaScript 함수를 정의합니다.

> > 주의점!!

- React 컴포넌트는 일반 JavaScript 함수이지만, 이름은 반드시 대문자로 시작해야 합니다.
- 그렇지 않으면 작동하지 않습니다!

> 3단계 : 마크업 추가

- 컴포넌트는 src 및 alt 속성을 가진 <img /> 태그를 반환합니다.
- <img />는 HTML처럼 작성되지만, 실제로는 JavaScript입니다!
- 이 구문은 JSX라고 하며, JavaScript 내에 마크업을 삽입할 수 있도록 합니다.
- 다음 컴포넌트처럼 반환문은 모두 한 줄에 작성할 수 있습니다.

```javascript
function Profile() {
  return <img src="https://i.imgur.com/MK3eW3Am.jpg" alt="Katherine Johnson" />;
}
```

- 하지만 마크업이 모두 return 키워드와 같은 줄에 있지 않으면 괄호 쌍으로 묶어야 합니다.

```javascript
return (
  <div>
    <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  </div>
);
```

> > 주의점!!

- 괄호가 없으면 return 뒤에 있는 모든 코드는 무시됩니다!

### 컴포넌트 사용하기

- 이제 Profile 컴포넌트를 정의했으므로 다른 컴포넌트 안에 중첩할 수 있습니다.
- 예를 들어, 여러 Profile 컴포넌트를 사용하는 Gallery 컴포넌트를 내보낼 수 있습니다.

App.js

```javascript
function Profile() {
  return <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />;
}

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

- 브라우저에서 보는 것
  대소문자의 차이에 주목하세요
  - <section>은 소문자로 시작하므로 React는 HTML 태그를 참조한다는 것을 알 수 있습니다.
  - <Profile />은 대문자 P로 시작하므로 React는 Profile이라는 컴포넌트를 사용하려고 한다는 것을 알 수 있습니다.
  - 그리고 Profile은 더 많은 HTML을 포함합니다: <img />. 최종적으로 브라우저에서 보는 것은 다음과 같습니다.

<section>
  <h1>Amazing scientists</h1>
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
  <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />
</section>

- 컴포넌트 중첩 및 구성

  - 컴포넌트는 일반적인 JavaScript 함수이므로 여러 컴포넌트를 같은 파일에 보관할 수 있습니다.
  - 이는 컴포넌트가 비교적 작거나 서로 밀접하게 관련되어 있을 때 편리합니다.
  - 이 파일이 너무 복잡해지면 언제든지 Profile을 별도의 파일로 이동할 수 있습니다.
  - 이 방법에 대한 자세한 내용은 곧 imports 페이지에서 살펴보겠습니다.

  - Profile 컴포넌트는 Gallery 내에서 여러 번 렌더링되므로 Gallery를 부모 컴포넌트라고 할 수 있으며,
  - 각 Profile을 "자식"으로 렌더링합니다.
  - 컴포넌트를 한 번 정의하면 원하는 만큼 여러 위치에서 사용할 수 있습니다.
    이것이 React의 마법 중 하나입니다.

> > 주의점!!

- 컴포넌트는 다른 컴포넌트를 렌더링할 수 있지만, 해당 정의를 중첩해서는 안 됩니다.

```javascript
export default function Gallery() {
  // 🔴 절대로 다른 컴포넌트 내부에 컴포넌트를 정의해서는 안됩니다!
  function Profile() {
    // ...
  }
  // ...
}
```

- 위 스니펫은 매우 느리고 버그를 발생시킵니다. 대신 모든 컴포넌트를 최상위 레벨로 정의합니다.

```javascript
export default function Gallery() {
  // ...
}

// ✅ 최상위 레벨에서 컴포넌트를 선언합니다
function Profile() {
  // ...
}
```

- 자식 구성 요소에 부모로부터 데이터가 필요한 경우, 중첩 정의 대신 props를 통해 데이터를 전달합니다.

```javascript
// 부모 컴포넌트
function App() {
  return <Welcome name="Sara" />; // 'name' 이라는 props를 자식 컴포넌트에 전달
}

// 자식 컴포넌트
function Welcome(props) {
  // props 객체로 {name: "Sara"}를 받음
  return <h1>Hello, {props.name}</h1>; // props.name을 사용해 "Hello, Sara" 렌더링
}
```

### 요약(Recap)

- React를 사용하면 앱에서 재사용 가능한 UI 요소인 컴포넌트를 만들 수 있습니다.
- React 앱에서 모든 UI는 컴포넌트입니다.
- React 컴포넌트는 일반 JavaScript 함수와 같지만, 다음과 같은 차이점이 있습니다.
  1. 컴포넌트 이름은 항상 대문자로 시작합니다.
  2. JSX 마크업을 반환합니다.

## 컴포넌트 가져오기(import) 및 내보내기(Export) [Importing and Exporting Components](https://react.dev/learn/importing-and-exporting-components)

- 컴포넌트의 매력은 재사용성에 있습니다.
- 다른 컴포넌트로 구성된 컴포넌트를 만들 수 있습니다.
- 하지만 컴포넌트를 중첩하는 횟수가 많아질수록 여러 파일로 분할하는 것이 더 합리적입니다.
- 이렇게 하면 파일을 쉽게 검색하고 더 많은 곳에서 컴포넌트를 재사용할 수 있습니다.

### Root 컴포넌트 파일

- 첫 번째 컴포넌트에서는 프로필 컴포넌트와 이를 렌더링하는 갤러리 컴포넌트를 만들었습니다.

```javascript
function Profile() {
  return <img src="https://i.imgur.com/MK3eW3As.jpg" alt="Katherine Johnson" />;
}

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

- 현재 위 예제에서 컴포넌트들은 App.js라는 root 컴포넌트 파일에 있습니다.
- 하지만 설정에 따라 root 컴포넌트가 다른 파일에 위치할 수도 있습니다.
- Next.js와 같이 파일 기반 라우팅을 사용하는 프레임워크를 사용하는 경우, 페이지별로 root 컴포넌트가 다를 수 있습니다.

### 컴포넌트 내보내기(Export)와 가져오기(import)

나중에 랜딩 화면을 변경하여 과학 서적 목록을 추가하거나 모든 프로필을 다른 곳에 배치하고 싶다면 어떻게 해야 할까요? Gallery와 Profile을 Root 컴포넌트 파일에서 옮기는 것이 좋습니다. 이렇게 하면 모듈성이 높아지고 다른 파일에서 재사용성이 높아집니다. 컴포넌트를 옮기는 단계는 세 가지입니다.

1. 컴포넌트를 추가할 새 JS 파일을 만듭니다.
2. 해당 파일에서 함수형 컴포넌트를 내보냅니다(기본 내보내기(Export) 또는 명명된 내보내기(Export) 사용).
3. 컴포넌트를 사용할 파일에서 가져옵니다(기본 내보내기(Export) 또는 명명된 내보내기(Export)를 가져오는 방법 사용).

여기서 Profile과 Gallery는 App.js 파일에서 Gallery.js라는 새 파일로 옮겨졌습니다.
이제 App.js 파일을 변경하여 Gallery.js 파일에서 Gallery를 가져올 수 있습니다.

App.js

```javascript
import Gallery from './Gallery.js';

export default function App() {
  return <Gallery />;
}
```

Gallery.js

```javascript
function Profile() {
  return <img src="https://i.imgur.com/QIrZWGIs.jpg" alt="Alan L. Hart" />;
}

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

이 예제가 두 개의 컴포넌트 파일로 어떻게 나뉘었는지 확인할 수 있습니다.

1. App.js:

- 기본 방식으로 Gallery.js에서 Gallery를 가져오기(import) 합니다.
- Root App 컴포넌트를 기본 방식으로 내보내기(Export) 합니다.

2. Gallery.js:

- Profile 컴포넌트를 정의하고 동일한 파일 내에서만 사용되어 내보내지(Export)지 않습니다.
- 기본 방식으로 Gallery 컴포넌트를 내보내기(Export) 합니다.

> > 참고!!

- 다음과 같이 .js 파일 확장자가 없는 파일이 있을 수 있습니다.

```javascript
import Gallery from './Gallery';
```

- React에서는 './Gallery.js'나 './Gallery' 모두 작동하지만, 전자가 native ES Modules 작동 방식에 더 가깝습니다.

### 같은 파일에서 여러 컴포넌트를 import 하거나 export 하는 방법

- 갤러리 대신 하나의 Profile 컴포넌트만 표시하려면 어떻게 해야 하나요?
- Profile 컴포넌트도 내보낼 수 있습니다.
- 하지만 Gallery.js에는 이미 기본 내보내기가 있으므로 두 개의 기본 내보내기를 사용할 수 없습니다.
- 기본 내보내기를 사용하여 새 파일을 만들거나 Profile에 명명된 내보내기를 추가할 수 있습니다
- 파일은 단 하나의 default export만 사용할 수 있지만 named export는 여러 번 사용할 수 있습니다.

> > 참고!!

- 기본 내보내기와 명명된 내보내기 간의 잠재적인 혼동을 줄이기 위해 일부 팀은 하나의 스타일(기본 또는 명명된 스타일)만 사용하거나 단일 파일에 두 스타일을 혼용하지 않아야 합니다.
- 자신에게 가장 적합한 방식을 선택하세요!

- 먼저, 명명된 내보내기(named export)를 사용하여 Gallery.js에서 Profile 컴포넌트를 내보냅니다(기본 키워드 없음).

```javascript
export function Profile() {
  // ...
}
```

- 그런 다음 명명된 가져오기(중괄호 포함)를 사용하여 Gallery.js에서 Profile 컴포넌트를 App.js로 가져옵니다.

```javascript
import { Profile } from './Gallery.js';
}
```

- 마지막으로 App 컴포넌트에서 <Profile />을 렌더링합니다.

```javascript
export default function App() {
  return <Profile />;
}
```

- 이제 Gallery.js에는 기본 Gallery 내보내기와 이름이 명명된 Profile 내보내기, 두 가지 내보내기가 있습니다.
- App.js는 두 컴포넌트를 import 해서 사용합니다. 다음 예제에서 <Profile />과 <Gallery />로 편집했다가 다시 원래대로 수정해 보세요.

App.js

```javascript
import Gallery from './Gallery.js';
import { Profile } from './Gallery.js';

export default function App() {
  return <Profile />;
}
```

Gallery.js

```javascript
export function Profile() {
  return <img src="https://i.imgur.com/QIrZWGIs.jpg" alt="Alan L. Hart" />;
}

export default function Gallery() {
  return (
    <section>
      <h1>Amazing scientists</h1>
      <Profile />
      <Profile />
      <Profile />
    </section>
  );
}
```

- 이제 기본 내보내기와 명명된 내보내기를 함께 사용할 수 있습니다.

  - Gallery.js:

    1. Profile 컴포넌트를 Profile이라는 명명된 내보내기로 내보냅니다.
    2. Gallery 컴포넌트를 기본 내보내기로 내보냅니다.

  - App.js:
    1. Gallery.js에서 Profile 컴포넌트를 Profile이라는 명명된 가져오기로 가져옵니다.
    2. Gallery.js에서 Gallery 컴포넌트를 기본 가져오기로 가져옵니다.
    3. root App 컴포넌트를 기본 내보내기로 내보냅니다.

- 컴포넌트를 가져오거나 내보내는 방법
- 기본 또는 명명된 가져오기와 내보내기를 사용하는 시기
- 같은 파일에서 여러 컴포넌트를 내보내는 방법

## JSX로 마크업 작성 [Writing Markup with JSX](https://react.dev/learn/writing-markup-with-jsx)

- JSX는 JavaScript를 확장한 문법으로, JavaScript 파일을 HTML과 비슷하게 마크업을 작성할 수 있습니다.
- 컴포넌트를 작성하는 다른 방법도 있지만, 대부분의 React 개발자는 JSX의 간결함을 선호하며 대부분의 코드 베이스에서 JSX를 사용합니다.

### JSX: JavaScript에 마크업 넣기

- 웹은 HTML, CSS, JavaScript로 구축되었습니다.
- 수년 동안 웹 개발자들은 HTML에 콘텐츠를, CSS로 디자인을, JavaScript로 로직을 각각 별도의 파일로 관리해 왔습니다!
- 콘텐츠는 HTML 내부에 마크업되었고, 페이지 로직은 JavaScript에 별도로 저장되었습니다.

- 하지만 웹이 더욱 인터랙티브해지면서 로직이 콘텐츠를 결정하는 경우가 되었습니다.
- JavaScript가 HTML을 담당하게 된 것입니다!
- 이것이 React에서 렌더링 로직과 마크업이 같은 위치, 즉 컴포넌트에 함께 존재하는 이유입니다.

- 버튼의 렌더링 로직과 버튼의 마크업을 함께 유지하면 모든 편집 과정에서 서로 동기화됩니다.
- 반대로, 버튼의 마크업과 사이드바의 마크업처럼 관련 없는 세부 사항은 서로 분리되어 있으므로, 각각 개별적으로 변경하는 것이 더 안전합니다.

- 각 React 컴포넌트는 React가 브라우저에 렌더링하는 마크업을 포함할 수 있는 JavaScript 함수입니다.
- React 컴포넌트는 JSX라는 확장된 구문을 사용하여 해당 마크업을 표현합니다.
- JSX는 HTML과 매우 유사하지만, HTML보다 약간 더 엄격하고 동적 정보를 표시할 수 있습니다.
- JSX를 이해하는 가장 좋은 방법은 일부 HTML 마크업을 JSX 마크업으로 변환해 보는 것입니다.

> > 참고!!

- JSX와 React는 별개의 개념입니다.
- 함께 사용되는 경우가 많지만, 서로 독립적으로 사용할 수도 있습니다.
- JSX는 확장된 문법인 반면, React는 JavaScript 라이브러리입니다.

### HTML을 JSX로 변환하기

- (완벽하게 유효한) HTML이 있다고 가정해 보겠습니다.

<h1>Hedy Lamarr's Todos</h1>
<img 
  src="https://i.imgur.com/yXOvdOSs.jpg" 
  alt="Hedy Lamarr" 
  class="photo"
>
<ul>
    <li>Invent new traffic lights
    <li>Rehearse a movie scene
    <li>Improve the spectrum technology
</ul>

- 그리고 이것을 컴포넌트에 넣고 싶습니다:

```javascript
export default function TodoList() {
  return (
    // ???
  )
}
```

- 이 코드를 그대로 복사해서 붙여넣으면 작동하지 않습니다.

App.js

```javascript
export default function TodoList() {
  return (
    // 이것은 동작하지 않습니다!
    <h1>Hedy Lamarr's Todos</h1>
    <img
      src="https://i.imgur.com/yXOvdOSs.jpg"
      alt="Hedy Lamarr"
      class="photo"
    >
    <ul>
      <li>Invent new traffic lights
      <li>Rehearse a movie scene
      <li>Improve the spectrum technology
    </ul>
  );
}
```

- JSX는 HTML보다 더 엄격하고 규칙이 더 많기 때문입니다!

### JSX의 규칙

1. 단일 루트 엘리먼트로 반환하기
   포넌트에서 여러 엘리먼트를 반환하려면 단일 부모 태그로 요소를 감싸야 합니다.

- 예를 들어, <div>를 사용할 수 있습니다.
<div>
  <h1>Hedy Lamarr's Todos</h1>
  <img 
    src="https://i.imgur.com/yXOvdOSs.jpg" 
    alt="Hedy Lamarr" 
    class="photo"
  >
  <ul>
    ...
  </ul>
</div>

- 마크업에 추가 <div>를 추가하고 싶지 않다면 대신 <> 및 </>를 작성할 수 있습니다.

```javascript
<>
  <h1>Hedy Lamarr's Todos</h1>
  <img
    src="https://i.imgur.com/yXOvdOSs.jpg"
    alt="Hedy Lamarr"
    class="photo"
  >
  <ul>
    ...
  </ul>
</>
```

- 이 빈 태그를 '프래그먼트'라고 합니다.
- 프래그먼트를 사용하면 브라우저 HTML 트리에 아무런 흔적도 남기지 않고 여러 요소를 그룹화할 수 있습니다.

- 왜 여러 개의 JSX 태그를 하나로 래핑해야 하나요?
  - JSX는 HTML처럼 보이지만, 실제로는 일반 JavaScript 객체로 변환됩니다.
  - 함수에서 두 객체를 반환하려면 배열로 감싸야 합니다.
  - 이는 두 개의 JSX 태그를 다른 태그나 프래그먼트로 감싸지 않고는 반환할 수 없습니다.

2. 모든 태그 닫기

- JSX는 태그를 명시적으로 닫아야 합니다.
- <img>와 같이 스스로 닫는 태그는 <img />로 작성해야 하고,
- <li>oranges와 같이 감싸는 태그는 <li>oranges</li>로 작성해야 합니다.

- 다음과 같이 Hedy Lamarr의 이미지와 목록 항목은 다음과 같이 닫혀 있습니다.

<>
<img
    src="https://i.imgur.com/yXOvdOSs.jpg"
    alt="Hedy Lamarr"
    class="photo"
   />

  <ul>
    <li>Invent new traffic lights</li>
    <li>Rehearse a movie scene</li>
    <li>Improve the spectrum technology</li>
  </ul>
</>

3. 대부분 camelCase로 작성하세요!

- JSX는 JavaScript로 변환되고, JSX로 작성된 속성은 JavaScript 객체의 키가 됩니다.
- 컴포넌트에서 이러한 속성을 변수로 읽어들이고 싶을 경우가 많습니다.
- 하지만 JavaScript는 변수명에 제한이 있습니다.
- 예를 들어, 변수 이름에는 대시(-)를 포함하거나 class와 같은 예약어를 사용할 수 없습니다.

- 이것이 React에서 많은 HTML 및 SVG 속성을 camelCase로 작성하는 이유입니다.
- 예를 들어, stroke-width 대신 strokeWidth를 사용합니다.
- class는 예약어이므로 React에서는 해당 DOM 속성의 이름을 따서 className으로 작성합니다.

<img
  src="https://i.imgur.com/yXOvdOSs.jpg"
  alt="Hedy Lamarr"
  className="photo"
/>

- 이러한 모든 속성은 DOM 컴포넌트 속성 목록에서 확인할 수 있습니다.
- 잘못 입력하더라도 걱정하지 마세요. React가 브라우저 콘솔에 수정 가능한 메시지로 알려줍니다.

> > 주의점!

- 역사적인 이유로 aria-_ 및 data-_ 속성은 HTML에서처럼 대시(-)를 사용하여 작성합니다.

4. 전문가 팁: JSX 변환기 사용

- 기존 마크업의 모든 속성을 변환하는 것은 번거로울 수 있습니다!
- 기존 HTML과 SVG를 JSX로 변환하려면 변환기를 사용하는 것이 좋습니다.
- 변환기는 실무에서 매우 유용하지만, JSX를 직접 편하게 작성할 수 있도록 진행 상황을 이해하는 것이 좋습니다.

```javascript
      />

        <li>Invent new traffic lights</li>
        <li>Rehearse a movie scene</li>
        <li>Improve the spectrum technology</li>

    </>
  );
}
```

- React 컴포넌트는 서로 관련이 있기 때문에 렌더링 로직을 마크업과 함께 그룹화합니다.
- JSX는 HTML과 유사하지만 몇 가지 차이점이 있습니다. 필요한 경우 변환기를 사용할 수 있습니다.
- 오류 메시지는 종종 마크업을 수정하는 올바른 방향을 알려줍니다.

## 중괄호를 사용한 JSX JavaScript

> JSX를 사용하면 렌더링 로직과 콘텐츠를 같은 위치에 유지하면서 JavaScript 파일 내에 HTML과 유사한 마크업을 작성할 수 있습니다.
> 경우에 따라 해당 마크업 내에 JavaScript 로직을 추가하거나 동적 속성을 참조해야 할 수 있습니다.
> 이 경우 JSX에 중괄호('{}')를 사용하여 JavaScript를 사용할 수 있습니다.

- 따옴표로 문자열 전달하기
  - JSX에 문자열 속성을 전달하려면 작은따옴표나 큰따옴표로 묶어야 합니다.

App.js

```javascript
export default function Avatar() {
  return (
    <img
      className="avatar"
      src="https://i.imgur.com/7vQD0fPs.jpg"
      alt="Gregorio Y. Zara"
    />
  );
}
```

- 여기서는 "https://i.imgur.com/7vQD0fPs.jpg"와 "Gregorio Y. Zara"가 문자열로 전달되고 있습니다.

- 하지만 src 또는 alt 텍스트를 동적으로 지정하려면 어떻게 해야 할까요?
- JavaScript에서 "와"를 {와}로 바꿔 값을 사용할 수 있습니다.

App.js

```javascript
export default function Avatar() {
  const avatar = 'https://i.imgur.com/7vQD0fPs.jpg';
  const description = 'Gregorio Y. Zara';
  return <img className="avatar" src={avatar} alt={description} />;
}
```

- 이미지를 둥글게 만드는 "avatar" CSS 클래스 이름을 지정하는
  className="avatar"와, avatar라는 JavaScript 변수의 값을 읽어오는 src={avatar}의 차이점을 눈여겨보세요.
  중괄호를 사용하면 마크업에서 바로 JavaScript를 사용할 수 있기 때문입니다!

### 중괄호 사용: JavaScript 세계로 통하는 창

> JSX는 JavaScript를 작성하는 특별한 방법입니다.
> 중괄호 { }를 사용하여 JavaScript를 JSX 내부에서 사용할 수 있습니다.
> 아래 예제는 먼저 과학자의 이름인 name을 선언한 다음, <h1> 안에 중괄호를 사용하여 해당 이름을 삽입합니다.

App.js

```javascript
export default function TodoList() {
  const name = 'Gregorio Y. Zara';
  return <h1>{name}'s To Do List</h1>;
}
```

- 이름 값을 'Gregorio Y. Zara'에서 'Hedy Lamarr'로 변경해 보세요.
- 목록 제목이 어떻게 바뀌는지 보이시나요?
- formatDate()와 같은 함수 호출을 포함하여 모든 JavaScript 표현식은 중괄호 안에서 작동합니다.

App.js

```javascript
const today = new Date();

function formatDate(date) {
  return new Intl.DateTimeFormat('en-US', { weekday: 'long' }).format(date);
}

export default function TodoList() {
  return <h1>To Do List for {formatDate(today)}</h1>;
}
```
